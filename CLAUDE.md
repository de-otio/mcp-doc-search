# Claude Code Instructions

## Git Commits

- **Always run `npm run lint` (or fix linting issues) before committing code.**
- Run relevant tests (`npm test`) before creating pull requests.
- Use descriptive commit messages following the existing commit style.
- Do not force-push to shared branches (main, dev).

## Code Quality

- Prefer editing existing files over creating new ones.
- Use the dedicated tools (Read, Edit, Write, Glob, Grep) instead of bash equivalents.
- Keep solutions focused and avoid over-engineering.

## Communication

- Be concise and use markdown formatting.
- Use relative file paths with markdown links: `[filename.ts](src/filename.ts)`
